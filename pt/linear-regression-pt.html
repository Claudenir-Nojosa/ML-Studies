Linear Regression e Gradient Descent - Guia Completo

1. Supervised Learning

O processo do Supervised Learning consiste em pegar o Training Set e alimentar esses dados para o nosso Learning Algorithm. O trabalho do algoritmo nos apresentar uma funo que faz predies.

Por conveno, essa funo chamada de hypothesis (hiptese). O trabalho do hypothesis pegar informaes (features) que ele no viu ainda e estimar o output corretamente.

2. Notao e Conceitos Fundamentais

Notao do Dataset

  • $m$ = nmero de training examples (nmero de linhas na tabela)
  • $n$ = nmero de features (variveis de entrada)
  • $x$ = inputs
  • $y$ = output / target value (o que queremos prever)
  • $(x, y)$ = um training example
  • $(x^{(i)}, y^{(i)})$ = o i-simo training example

Parmetros

$\theta$ (theta) so chamados de parmetros (parameters). O trabalho do learning algorithm escolher os parmetros $\theta$ que permitem fazer predies boas.

3. Dataset de Exemplo

Vamos usar um dataset super simples com apenas 3 casas para entender os clculos:

Size (m) Preo (mil R$)
50 150
80 200
110 250

Ento temos:

  • $m = 3$ (3 training examples)
  • $n = 1$ (1 feature: size)
  • $x^{(1)} = 50, \quad y^{(1)} = 150$
  • $x^{(2)} = 80, \quad y^{(2)} = 200$
  • $x^{(3)} = 110, \quad y^{(3)} = 250$

4. Hypothesis (Nossa Reta)

Representao Bsica

Em Linear Regression com uma feature, o hypothesis representado como:

$$h(x) = \theta_0 + \theta_1 x$$

No exemplo acima, $\theta_0$ = 1.5 e $\theta_1$ = 0

Observamos que se assemelha bastante com uma equao do primeiro grau, onfe $f(x) = b + ax$

Onde:

  • $\theta_0$ = intercepto (onde a reta corta o eixo y)
  • $\theta_1$ = inclinao (coeficiente angular)
  • $x$ = input (size), ou seja, o que queremos usar para fazer a predio

Exemplo numrico: Se $\theta_0 = 50$ e $\theta_1 = 2$, ento:

$$h(x) = 50 + 2x$$

Para uma casa de 80m: $$h(80) = 50 + 2(80) = 50 + 160 = 210 \text{ mil R\$}$$

Mltiplas Features

Quando temos mais de uma feature (varivel), como o nmero de quartos por exemplo:

$$h(x) = \theta_0 + \theta_1x_1 + \theta_2x_2$$

Onde:

  • $x_1$ = size
  • $x_2$ = #bedrooms

5. Cost Function - Medindo o Erro

A Cost Function $J(\theta)$ mede o quo longe nossas predies esto dos valores reais. a diferena entre o valor real dos dados ($y$) menos o valor previsto pela reta ($h(x)$), ao quadrado, somando em todos os data points:

$$J(\theta_0, \theta_1) = \frac{1}{2m} \sum_{i=1}^{m} (h(x^{(i)}) - y^{(i)})^2$$

A inteno fazer com que essa diferena seja a menor possvel.

Frmula para descobrir o $\theta_0$ e $\theta_1$

$$ \theta_1 = \frac { \sum_{i=1}^{m} \left(x^{(i)} - \bar{x}\right) \left(y^{(i)} - \bar{y}\right) } { \sum_{i=1}^{m} \left(x^{(i)} - \bar{x}\right)^2 } $$

$$ \theta_0 = \bar{y} - \theta_1 \bar{x} $$

Exemplo de Clculo Passo a Passo

Vamos calcular $J(\theta_0, \theta_1)$ para $\theta_0 = 50$ e $\theta_1 = 2$ usando nosso dataset:

Passo 1: Calcular as predies $h(x^{(i)})$

  • $h(x^{(1)}) = 50 + 2(50) = 150$
  • $h(x^{(2)}) = 50 + 2(80) = 210$
  • $h(x^{(3)}) = 50 + 2(110) = 270$

Passo 2: Calcular os erros $(h(x^{(i)}) - y^{(i)})$

  • Erro 1: $150 - 150 = 0$
  • Erro 2: $210 - 200 = 10$
  • Erro 3: $270 - 250 = 20$

Passo 3: Elevar ao quadrado

  • Erro 1: $0^2 = 0$
  • Erro 2: $10^2 = 100$
  • Erro 3: $20^2 = 400$

Passo 4: Somar e dividir por $2m$

$$J(50, 2) = \frac{1}{2(3)}(0 + 100 + 400) = \frac{500}{6} = 83.33$$

6. Gradient Descent - Encontrando os Melhores Parmetros

Para minimizar $J(\theta)$, utilizamos Gradient Descent. Basicamente, a gente olha em 360 e procura valores de $\theta_0$ e $\theta_1$ para encontrar o menor $J(\theta)$ o mais rpido possvel.

Algoritmo do Gradient Descent

Repetir at convergir:

$$\theta_j := \theta_j - \alpha \frac{\partial}{\partial \theta_j} J(\theta)$$

Onde:

  • $:=$ significa assignment (atribuio), ou seja, o valor atualizado por um novo
  • $\alpha$ o learning rate (taxa de aprendizado)
  • $\frac{\partial}{\partial \theta_j} J(\theta)$ a derivada parcial da cost function

Learning Rate

  • Na prtica, o learning rate geralmente definido como $\alpha = 0.01$
  • A derivada de uma funo define a direo do steepest descent (descida mais ngreme), ou seja, ir downhill o mais rpido possvel

Derivadas Parciais para Linear Regression

Para Linear Regression, as derivadas so:

$$\frac{\partial}{\partial \theta_0} J(\theta_0, \theta_1) = \frac{1}{m} \sum_{i=1}^{m} (h(x^{(i)}) - y^{(i)})$$

$$\frac{\partial}{\partial \theta_1} J(\theta_0, \theta_1) = \frac{1}{m} \sum_{i=1}^{m} (h(x^{(i)}) - y^{(i)}) \cdot x^{(i)}$$

Exemplo Numrico Detalhado

Vamos executar o Gradient Descent manualmente com nosso dataset!

Valores iniciais:

  • $\theta_0 = 0$
  • $\theta_1 = 0$
  • $\alpha = 0.01$ (learning rate)

Iterao 1

Passo 1: Calcular predies com $\theta_0 = 0, \theta_1 = 0$

  • $h(50) = 0 + 0(50) = 0$
  • $h(80) = 0 + 0(80) = 0$
  • $h(110) = 0 + 0(110) = 0$

Passo 2: Calcular erros

  • Erro 1: $0 - 150 = -150$
  • Erro 2: $0 - 200 = -200$
  • Erro 3: $0 - 250 = -250$

Passo 3: Calcular derivada para $\theta_0$

$$\frac{\partial J}{\partial \theta_0} = \frac{1}{3}(-150 - 200 - 250) = \frac{-600}{3} = -200$$

Passo 4: Calcular derivada para $\theta_1$

$$\frac{\partial J}{\partial \theta_1} = \frac{1}{3}[(-150)(50) + (-200)(80) + (-250)(110)]$$ $$= \frac{1}{3}[-7500 - 16000 - 27500] = \frac{-51000}{3} = -17000$$

Passo 5: Atualizar parmetros

$$\theta_0 := 0 - 0.01(-200) = 0 + 2 = 2$$ $$\theta_1 := 0 - 0.01(-17000) = 0 + 170 = 170$$

Agora temos: $\theta_0 = 2$ e $\theta_1 = 170$

Iterao 2

Passo 1: Calcular predies com $\theta_0 = 2, \theta_1 = 170$

  • $h(50) = 2 + 170(50) = 8502$
  • $h(80) = 2 + 170(80) = 13602$
  • $h(110) = 2 + 170(110) = 18702$

Passo 2: Calcular erros

  • Erro 1: $8502 - 150 = 8352$
  • Erro 2: $13602 - 200 = 13402$
  • Erro 3: $18702 - 250 = 18452$

Observe: Os valores esto muito longe! Isso acontece porque o $\alpha$ multiplicado por derivadas grandes faz "pulos" enormes. Com mais iteraes, os valores vo convergindo.

Com muitas iteraes, os valores vo convergindo at os valores timos!

7. Tipos de Gradient Descent

Batch Gradient Descent

Usa todos os dados em cada iterao. o que fizemos acima:

$$\theta_j := \theta_j - \alpha \frac{1}{m} \sum_{i=1}^{m} (h(x^{(i)}) - y^{(i)}) \cdot x_j^{(i)}$$

Cada passo do gradient descent requer percorrer o dataset inteiro. Isso no bom quando temos um dataset muito grande, pois fica muito demorado.

Stochastic Gradient Descent (SGD)

Para contornar o problema do Batch GD, existe o Stochastic Gradient Descent, que usa apenas 1 exemplo por vez, escolhido aleatoriamente:

$$\theta_j := \theta_j - \alpha (h(x^{(i)}) - y^{(i)}) \cdot x_j^{(i)}$$

O SGD pega uma casa aleatria, prev o preo, e vai ajustando os parmetros com outra casa aleatria, testando iterativamente at realmente encontrar ou chegar bem prximo ao global optimum.

Vantagem: Muito mais rpido para datasets grandes!

8. Resultado Final

Aps muitas iteraes (cerca de 100), o Gradient Descent encontra os valores timos. Para nosso dataset, os valores ideais so aproximadamente:

$$\theta_0 \approx 83.33$$ $$\theta_1 \approx 1.67$$

Ento nossa reta final :

$$h(x) = 83.33 + 1.67x$$

Exemplo de predio: Para prever o preo de uma casa de 90m:

$$h(90) = 83.33 + 1.67(90) = 83.33 + 150.3 = 233.63 \text{ mil R\$}$$

def cost_function(X, y, theta0, theta1):
    """
    Calcula J(theta0, theta1) = (1/2m) * sum((h(x) - y))
    """
    m = len(X)
    predictions = hypothesis(X, theta0, theta1)
    errors = predictions - y
    squared_errors = errors ** 2
    cost = (1 / (2 * m)) * np.sum(squared_errors)
    return cost

# ============================================
# 4. GRADIENT DESCENT - IMPLEMENTAO
# ============================================
print("=" * 60)
print("4. GRADIENT DESCENT")
print("=" * 60)


def compute_gradients(X, y, theta0, theta1):
    """
    Calcula as derivadas parciais:
    J/ = (1/m) * sum(h(x) - y)
    J/ = (1/m) * sum((h(x) - y) * x)
    """
    m = len(X)
    predictions = hypothesis(X, theta0, theta1)
    errors = predictions - y

    d_theta0 = (1/m) * np.sum(errors)
    d_theta1 = (1/m) * np.sum(errors * X)

    return d_theta0, d_theta1


def gradient_descent(X, y, theta0_init, theta1_init, alpha, iterations, verbose=True):
    """
    Executa Gradient Descent (BATCH)

     :=  -  * (J/)
    """
    theta0 = theta0_init
    theta1 = theta1_init
    m = len(X)
    history = []

    if verbose:
        print(f"Valores iniciais:  = {theta0},  = {theta1}")
        print(f"Learning rate (): {alpha}")
        print(f"Iteraes: {iterations}\n")

    for i in range(iterations):
        # Calcular predies
        predictions = hypothesis(X, theta0, theta1)

        # Calcular erros
        errors = predictions - y

        # Calcular derivadas parciais (gradientes)
        d_theta0, d_theta1 = compute_gradients(X, y, theta0, theta1)

        # Atualizar parmetros usando a regra do gradient descent
        theta0 = theta0 - alpha * d_theta0
        theta1 = theta1 - alpha * d_theta1

        # Calcular cost
        cost = cost_function(X, y, theta0, theta1)

        # Salvar no histrico
        history.append((theta0, theta1, cost))

        # Mostrar progresso detalhado nas primeiras 2 iteraes
        if verbose and i < 2:
            print(f"{'' * 50}")
            print(f"ITERAO {i+1}")
            print(f"{'' * 50}")
            print(
                f"Predies h(x): [{predictions[0]:.2f}, {predictions[1]:.2f}, {predictions[2]:.2f}]")
            print(
                f"Erros (h(x) - y): [{errors[0]:.2f}, {errors[1]:.2f}, {errors[2]:.2f}]")
            print(f"\nDerivadas parciais:")
            print(
                f"  J/ = (1/{m})  ({errors[0]:.2f} + {errors[1]:.2f} + {errors[2]:.2f})")
            print(f"         = {d_theta0:.4f}")
            print(
                f"\n  J/ = (1/{m})  ({errors[0]:.2f}{X[0]} + {errors[1]:.2f}{X[1]} + {errors[2]:.2f}{X[2]})")
            print(f"         = {d_theta1:.4f}")
            print(f"\nAtualizao dos parmetros:")
            prev_theta0 = theta0 + alpha * d_theta0
            prev_theta1 = theta1 + alpha * d_theta1
            print(
                f"   := {prev_theta0:.4f} - {alpha}  {d_theta0:.4f} = {theta0:.4f}")
            print(
                f"   := {prev_theta1:.4f} - {alpha}  {d_theta1:.4f} = {theta1:.4f}")
            print(f"\nCost J() = {cost:.4f}\n")

    return theta0, theta1, history

def stochastic_gradient_descent(X, y, theta0_init, theta1_init, alpha, iterations):
    """
    SGD: Atualiza  usando apenas 1 exemplo aleatrio por iterao
    """
    theta0 = theta0_init
    theta1 = theta1_init
    m = len(X)
    history = []

    for i in range(iterations):
        # Escolher exemplo aleatrio
        idx = np.random.randint(0, m)
        x_i = X[idx]
        y_i = y[idx]

        # Predio e erro para este exemplo
        prediction = hypothesis(x_i, theta0, theta1)
        error = prediction - y_i

        # Atualizar usando apenas este exemplo
        theta0 = theta0 - alpha * error
        theta1 = theta1 - alpha * error * x_i

        # Calcular cost com todos os dados (para monitoramento)
        cost = cost_function(X, y, theta0, theta1)
        history.append((theta0, theta1, cost))

    return theta0, theta1, history
