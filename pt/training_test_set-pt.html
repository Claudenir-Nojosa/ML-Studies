Train, Validation e Test Split

Introduo

Aps implementar um modelo de Machine Learning, surge uma questo fundamental: como sabemos se nosso modelo realmente funciona bem? Treinar um modelo e test-lo nos mesmos dados no suficiente, pois isso no garante que ele funcionar bem com dados novos que nunca viu antes.

A diviso dos dados em conjuntos de treinamento, validao e teste uma prtica essencial para avaliar corretamente o desempenho de modelos de aprendizado de mquina.

O Problema: Overfitting vs Underfitting

Overfitting (Sobreajuste)

Overfitting ocorre quando o modelo "decora" os dados de treinamento, capturando no apenas os padres reais, mas tambm o rudo e peculiaridades especficas daqueles dados.

Caractersticas:

  • Performance excelente nos dados de treinamento
  • Performance ruim em dados novos
  • O modelo muito especfico e no generaliza

Analogia: como um aluno que decora as respostas de provas antigas, mas no entende os conceitos. Ele vai bem nas provas antigas, mas falha em questes novas.

Underfitting (Subajuste)

Underfitting ocorre quando o modelo muito simples e no consegue capturar os padres presentes nos dados.

Caractersticas:

  • Performance ruim nos dados de treinamento
  • Performance ruim em dados novos
  • O modelo simples demais

Analogia: como um aluno que no estudou o suficiente e no consegue responder nem as questes bsicas.

O Equilbrio Ideal

O objetivo encontrar o ponto ideal onde o modelo:

  • Aprende os padres reais dos dados
  • Generaliza bem para dados novos
  • No decora peculiaridades especficas

Train/Test Split (Diviso Bsica)

Conceito

A diviso mais simples separa os dados em dois conjuntos:

  1. Training Set (Conjunto de Treinamento): ~70-80% dos dados

    • Usado para treinar o modelo
    • O modelo aprende os padres aqui

  2. Test Set (Conjunto de Teste): ~20-30% dos dados

    • Usado apenas para avaliar o modelo final
    • Simula dados "do mundo real" que o modelo nunca viu

Por Que Fazer Isso?

Sem diviso:

Treina no dataset  Testa no mesmo dataset  R = 0.99

Parece timo, mas enganoso! O modelo pode estar apenas decorando.

Com diviso:

Treina no training set  Testa no test set  R = 0.95

Agora temos uma medida real de como o modelo generaliza.

Propores Tpicas

Tamanho do Dataset Train Test
Pequeno (< 1000) 70% 30%
Mdio (1k-100k) 80% 20%
Grande (> 100k) 90% 10%

Regra geral: Quanto mais dados voc tem, menor a porcentagem necessria para teste.

Train/Validation/Test Split (Diviso Completa)

Conceito

Para projetos mais robustos, dividimos os dados em trs conjuntos:

  1. Training Set: ~60-70% dos dados

    • Usado para treinar o modelo
    • Ajusta os parmetros (w, b)

  2. Validation Set: ~15-20% dos dados

    • Usado para tunar hiperparmetros
    • Comparar diferentes modelos
    • Detectar overfitting durante o treinamento
    • No usado para treinar

  3. Test Set: ~15-20% dos dados

    • Usado apenas uma vez no final
    • Avaliao final e imparcial
    • Simula o desempenho em produo

Por Que Trs Conjuntos?

Problema com apenas Train/Test:

  • Se usarmos o test set para ajustar hiperparmetros, ele "vaza" informao (data leakage)
  • O test set deixa de ser imparcial
  • Podemos fazer overfitting no test set!

Soluo com Train/Validation/Test:

  • Validation usado para experimentao e ajustes
  • Test permanece "intocado" at o final
  • Temos uma avaliao verdadeiramente imparcial

Diviso Aleatria vs Estratificada

Diviso Aleatria

Seleciona exemplos aleatoriamente para cada conjunto.

np.random.shuffle(data)
train = data[:60%]
val = data[60%:80%]
test = data[80%:]

Quando usar:

  • Dados balanceados
  • Regresso (valores contnuos)
  • Datasets grandes

Diviso Estratificada

Mantm a mesma proporo de classes em todos os conjuntos.

Quando usar:

  • Classificao com classes desbalanceadas
  • Ex: 95% classe A, 5% classe B
  • Garante que train, val e test tenham ~95%/5%

Boas Prticas

O Que Fazer

  1. Dividir ANTES de qualquer processamento

    • Normalize depois da diviso
    • Evita "data leakage"

  2. Nunca treinar com dados de validao/teste

    • Validao apenas para avaliar
    • Teste apenas para avaliao final

  3. Manter test set "sagrado"

    • Use apenas UMA VEZ no final
    • No ajuste nada baseado nele

  4. Shuffle (embaralhar) os dados

    • Evita bias se dados esto ordenados

O Que NO Fazer

  1. Normalizar antes de dividir (causa data leakage)
  2. Usar test set para ajustar hiperparmetros
  3. Treinar com validation/test set
  4. Avaliar mltiplas vezes no test set
  5. Dividir dados temporais aleatoriamente (use diviso temporal)

Dados Temporais (Time Series)

Para dados com componente temporal (preos de aes, vendas ao longo do tempo), NO embaralhe!

Exemplo:

  • Train: Janeiro - Agosto (8 meses)
  • Validation: Setembro - Outubro (2 meses)
  • Test: Novembro - Dezembro (2 meses)

Por qu?

  • Em produo, voc sempre prev o futuro baseado no passado
  • Embaralhar cria "vazamento temporal" (model sees the future)

Resumo

Tipo do Dado Train Set Validation Set Test Set
Uso Treinar modelo Ajustar hiperparmetros Avaliao final
Frequncia Mltiplas vezes Mltiplas vezes Uma vez
Tamanho 60-80% 10-20% 10-20%
Pode treinar? Sim No No

import numpy as np

import matplotlib.pyplot as plt

from sklearn.model_selection import train_test_split

# ============================================================
# GERAO DOS DADOS (DATASET SINTTICO)
# ============================================================

# Fixa a semente para garantir reprodutibilidade
np.random.seed(42)

# Varivel independente X (50 pontos entre 0 e 10)
X = np.random.rand(50, 1) * 10

# Varivel dependente y
# Relao linear: y = 2.5x + 5 + rudo
y = 2.5 * X.ravel() + 5 + np.random.randn(50) * 3

print("\n CENRIO 1: SEM TRAIN/TEST SPLIT")

# Cria o modelo
model_sem_split = LinearRegression()

# Treina o modelo usando TODOS os dados
model_sem_split.fit(X, y)

# Faz previses nos mesmos dados usados no treino
y_pred_sem_split = model_sem_split.predict(X)

# Calcula o R usando treino = teste (problema!)
r2_sem_split = r2_score(y, y_pred_sem_split)

print(f"R = {r2_sem_split:.4f}")
print("  Avaliao enganosa: o modelo foi testado nos mesmos dados que treinou!\n")

# ============================================================
# CENRIO 2  COM TRAIN/TEST SPLIT (CORRETO)
# ============================================================

print("=" * 70)
print(" CENRIO 2: COM TRAIN/TEST SPLIT")
print("-" * 70)

# Divide os dados:
# 70% treino | 30% teste
X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.3, random_state=42
)

print(f"Training: {len(X_train)} exemplos | Test: {len(X_test)} exemplos\n")

# Cria o modelo
model_bom = LinearRegression()

# Treina APENAS com dados de treino
model_bom.fit(X_train, y_train)

# Previses no treino
y_train_pred = model_bom.predict(X_train)

# Previses em dados NUNCA vistos (teste)
y_test_pred = model_bom.predict(X_test)

# Avaliao correta
r2_train_bom = r2_score(y_train, y_train_pred)
r2_test_bom = r2_score(y_test, y_test_pred)

print(f"R Training: {r2_train_bom:.4f}")
print(f"R Test:     {r2_test_bom:.4f}")
print(f"Diferena:   {abs(r2_train_bom - r2_test_bom):.4f}")
print(" Modelo generaliza bem!\n")
